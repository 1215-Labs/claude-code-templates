# Audit Report Template

Use this template when generating the final alignment audit report during `/repo-audit` Phase 3.

## Variables

- `{REPO_NAME}` - Repository name
- `{DATE}` - Audit date (YYYY-MM-DD)
- `{EXECUTIVE_SUMMARY}` - Synthesized from all fork executive summaries
- `{DOCS_SCORE}`, `{DOCS_GRADE}`, `{DOCS_CRITICAL}` - Docs-to-code layer results
- `{INTERNAL_SCORE}`, `{INTERNAL_GRADE}`, `{INTERNAL_CRITICAL}` - Internal consistency results
- `{DEPLOY_SCORE}`, `{DEPLOY_GRADE}`, `{DEPLOY_CRITICAL}` - Code-to-deploy results (if applicable)
- `{OVERALL_SCORE}`, `{OVERALL_GRADE}`, `{OVERALL_CRITICAL}` - Weighted average
- `{GATING_NOTE}` - If gating issues exist: "Grade capped at C due to gating issues"

## Usage Notes

- Omit Layer 3 section entirely if code-to-deploy was not applicable
- The Fix First table should be sorted by severity (HIGH first) then effort (S first)
- Effort estimation: S = single edit, M = multiple files, L = structural change
- For verified claims, only show a representative sample (10-15 most important)
- For discrepancies, show ALL findings regardless of count
- The Evidence Index helps audit transparency — shows what was actually inspected
